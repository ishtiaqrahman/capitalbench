# Research Prompt Workflow

CapitalBench research uses three operator prompts before model submissions are
collected. These prompts generate audit artifacts and the final briefing; they
do not replace the mechanical model-input builder.

The effective model input is assembled later by `capitalbench build_prompt` from:

- `prompt.md`
- round metadata
- `briefing.md`, copied from Prompt 3's `final_briefing.md`
- `market_data/universe_trailing_returns.md`, generated by `capitalbench fetch-universe-performance`
- `options.yaml`

## Prompt 1: Market Fact Report

Goal: produce `market_fact_report.md`, an audit-only source report.

Required instructions:

- Gather only facts available at or before the research cutoff.
- Include source URLs, publisher names, publication dates, observation dates,
  and source-reported uncertainty in this audit-only report.
- Cover current macro releases, rates, index closes, market breadth, scheduled
  catalysts inside the scoring window, and any important cross-asset regime
  datapoints available from reliable sources.
- Treat mechanical market data as a separate CapitalBench artifact. Do not
  create a manually selected return table as if it were research evidence.
- If mechanical price context is referenced, refer to the complete generated
  artifact: `market_data/universe_trailing_returns.md`.
- Do not rank options, recommend allocations, or map facts to expected winners.

## Prompt 2: Briefing Audit Report

Goal: produce `briefing_audit_report.md`, an audit-only check of Prompt 1 and
Prompt 3 outputs.

Required checks:

- `final_briefing.md` contains no URLs, citations, source ledger, recommendations,
  rankings, subjective analysis, or affected-asset mapping.
- `final_briefing.md` does not contain a `Selected Mechanical Return Context`
  section or any manually selected subset of mechanical return rows.
- The generated full-universe price-context artifact exists when intended and
  covers every included option.
- The generated price-context artifact is sorted by option order, not by
  performance, and includes returns, benchmark-relative diagnostics, volatility,
  drawdown, path quality, 52-week position, and SPY beta/correlation when
  available.
- The prompt package explicitly states that price history is descriptive context,
  not a forecast.
- The final model input contains the price-context appendix exactly once.

## Prompt 3: Final Briefing

Goal: produce `final_briefing.md`, the only model-facing research artifact.

Required instructions:

- Write fixed factual datapoints only.
- Keep the briefing concise and balanced across broad asset areas.
- Include dates, values, publishers, statuses, forecasts labeled as forecasts,
  scheduled catalysts, and source-reported uncertainty.
- Do not include source URLs, source ledgers, recommendation language, rankings,
  scenario analysis, "why it matters" commentary, or affected-option mapping.
- Do not include selected mechanical return rows. The complete mechanical
  appendix is added by the prompt builder after the briefing.
- Include this neutrality sentence near the top:

```text
This briefing provides fixed factual datapoints only. It does not rank, recommend, analyze, or map facts to CapitalBench options. Inclusion, order, grouping, and row count are not evidence of expected return. A mechanical full-universe price, risk, and benchmark-relative appendix follows this briefing in the model input.
```

Use the same idea in the final neutrality statement. Do not call the appendix a
selected or ranked return table.
